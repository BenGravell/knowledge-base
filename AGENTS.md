# knowledge-base

A personal knowledge base of distilled notes, paper summaries, and explainers, published as a static site via MkDocs. Includes an embedding-based Mind Map and Streamlit tooling for browsing and generating entries.

## Key Commands

All commands run from the repo root unless noted.

```bash
# Install dependencies
poetry install

# Activate virtualenv
poetry shell
```

```bash
# From knowledge_base/
mkdocs serve        # dev server with live reload
mkdocs build        # build static site
mkdocs gh-deploy    # deploy to GitHub Pages
```

```bash
# Mind Map
python mind_map/generate_mind_map_data.py   # regenerate embeddings
python mind_map/preview_mind_map.py         # quick local preview

# Dev tools, for manual human use only
python scripts/audit_metadata.py            # audit entry metadata
streamlit run apps/analyzer_app.py          # browse / analyze entries
streamlit run apps/generator_app.py         # generate entries from arXiv
```

## Architecture

```text
knowledge-base/
├── knowledge_base/
│   ├── apps/          # Streamlit apps (analyzer, generator)
│   ├── docs/          # MkDocs markdown content (papers, explainers, templates)
│   ├── mind_map/      # Embedding + UMAP + graph generation scripts
│   ├── notebooks/     # Jupyter / marimo notebooks
│   ├── scripts/       # Audit and utility scripts
│   ├── mkdocs.yml     # MkDocs configuration
│   ├── site/          # Autogenerated site files. Do not edit these directly, as they will be overwritten.
│   └── config.py      # Shared config
├── todo/              # Paper and topic tracking lists
├── AGENTS.md          # Agent directives (this file)
├── CLAUDE.md          # Claude Code entry point (sources this file)
└── pyproject.toml     # Poetry project config
```

## Conventions

- Python `>=3.11, <3.14`; managed with **Poetry**
- MkDocs commands must be run from the `knowledge_base/` subdirectory
- New knowledge entries go under `knowledge_base/docs/` following the structure of existing files
- No test suite; verify changes by running `mkdocs build` and checking for warnings
